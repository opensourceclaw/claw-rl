## Changes

### 1. Chinese Feedback Pattern Support (Major Fix)

**Problem:** BinaryRLJudge and LLMPRMJudge couldn't properly recognize Chinese feedback like "感谢", "满意", "错误", "不要", etc.

**Solution:**
- Added 10+ missing Chinese patterns to POSITIVE_PATTERNS:
  - `感谢`, `感恩`, `谢了`, `谢谢您`, `满意`, `满足`, `认可`, `赞`, `点赞`, `准确`, `正确`
- Added 10+ missing Chinese patterns to NEGATIVE_PATTERNS:
  - `不满意`, `不好`, `错误`, `不要`, `拒绝`, `否定`, `算了`
- Optimized pattern matching order (negative patterns checked first)
- Updated `_get_rule_reason` to return Chinese-compatible reasons

**Test Results:**
- `test_binary_rl.py`: 33 passed → 36 passed ✅
- `test_llm_prm_judge.py`: 16 passed → 20 passed ✅
- `test_adaptive_mab.py`: 35 passed ✅

### 2. Enhanced Pattern Matching

Added support for more nuanced feedback:
- Continuation requests: `还有吗`, `继续`, `再来`
- Emoji reactions: 👍, 👌, ✅, 😊, 😄
- Mixed language: `太thank you了`

## Migration Notes

No breaking changes. This is a backward-compatible improvement.

**If you're using custom patterns**, you may want to review:
- Pattern matching now checks NEGATIVE_PATTERNS before POSITIVE_PATTERNS
- Some patterns like `不要` are now classified as negative
