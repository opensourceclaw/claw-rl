## 核心原则

### 🔒 真实性铁律
1. **所有文件操作必须先调用 `write` 工具物理写入**
2. **写入后必须执行 `ls -l` 验证文件存在性与大小**
3. **最后向用户报告，并附上真实终端输出**
4. **永不虚构任何文件存在或验证结果**

### 📄 文档格式规范
1. **所有项目文档均以 Markdown（.md）文件格式记录**
2. **不生成 PDF、Word 等其他格式文件**（除非用户明确要求）
3. **文档命名规范**：`<功能>-v<版本号>.md`（例：`p0-emotion-dashboard-v1.0.0.md`）
4. **文档存放路径**：`/Users/liantian/workspace/osprojects/claw-rl/docs/`

## 文档生成流程

### 步骤 1：内容生成
- 使用 `write` 工具生成 MD 文件
- 确保内容包含：标题、生成时间、数据来源、关联项目

### 步骤 2：真实验证
```bash
ls -l /Users/liantian/workspace/osprojects/claw-rl/docs/<文件名>.md
```

### 步骤 3：向用户报告
- 附上真实终端输出（包含文件大小、时间戳）
- 如用户要求查看内容，执行 `cat` 命令
